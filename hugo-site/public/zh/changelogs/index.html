    1.4.x 版本

    1.4.0 再访颜色

    功能

    • Bases 样式设置的改变不再是实验性的
    • 现在, 您可以为 ASCII 艺术和引号指定目标字体系列 (text, interface 或 monospace)
      • 为 ASCII 艺术, 我们建议使用类似等宽字体, 例如 Source Code Pro.
    • 现在, 您可以配置突出显示文本的颜色和不透明度
    • 修复工具提示
    • 添加了 “不在新选项卡中显示状态栏” 选项
      • 或者, 您可以在新选项卡中设置状态栏的不透明度. 默认值为 0.55.
    • 添加了 “Bases 卡片标签不透明度” 设置, 默认值为 0.55
    • 添加了 Typewriter 模式 (除活动行外全部变不透明), 不透明度可配置. 默认值为 0.55. 您也可以通过 cssclass 进行设置。
    • 现在, 当应用程序中未指定默认光标时, 您可以设置它
    • 现在, 您可以选择彩色标题时每个标题使用的颜色
    • 添加了 Bases 视图垂直和水平填充设置, 影响 Bases 文件内容. 两者的默认值均为 16px
    • 添加了 Bases 嵌入填充, 默认为 4px
    • 添加了与表头和单元格对齐相关的设置, 拆分了表头和单元格的现有设置
    • 增加了表格单元格字体大小设置, 将 “表头大小” 重命名为 “表头字体大小” 以更具体. 现在, 这两个选项都允许您从 UI 字体大小选项中进行选择
    • 添加了 “不显示内联标题” 设置, 具有 csslcass 等效的 no-inline-title
    • 添加了 “不显示视图标题” cssclass
    • 添加了移动部分和 “恢复 1.11 导航 UI” 设置, 恢复到旧的 vault 选择器, 导航到 1.11 之前的版本

    变化

    • 样式设置持续时间单位从 s 更改为 ms
      • 受影响的设置包括 alt 文件树、动画、弹出和抽认卡 callout、悬停前的状态隐藏和光标持续时间
    • 主题现在为一些衍生颜色使用 oklch 颜色空间
    • 将输入相关设置移动到 Editor > Input 下的独立部分
    • 由于版权和许可不明确, 将示例树 ASCII 更改为 Figlet 占位符
    • 更改了 ASCII 艺术文本对齐方式, 使其具有选项下拉列表, 而不是文本输入
    • 修改了社区项目背景颜色
    • 彩色标题不再出现在 PDF 导出输出中, 归功于 Fancy-a-Story 主题开发人员 Kapirklaa 的打印 (PDF 导出) 样式
    • 在 pl10k 模式中修复了设备操作细节, 添加了左侧填充设置. 现有的右侧填充设置的默认值已更改.

    修复

    • 修复工具提示和交互式强调色
    • 修复 pl10k 样式
    • 修复了替代链接样式, 它现在继承了现有的链接颜色变量并使用 oklch
    • 提高了主题中动画的性能
    • 在需要时定义超限行为
    • 清理并修复代码库的部分
    • pl10k 模式视图标题中的固定边界半径
    • 修复显示功能区时的垂直导航填充
    • Banner CSS 代码片段修补程序

    1.3.x 版本

    1.3.0 动画更新

    功能

    • 现在, 您可以在笔记与模态框等 UI 其他部分之间使用不同的动画集。
    • 新增淡入动画效果, 作为 Flex Max 模式的默认设置, 取代原有的从左至右滑动效果
    • 新增动画初始透明度和放大动画初始缩放比例的样式设置选项
    • 添加了主题中 Banner CSS 代码片段支持的修复
    • 在行间距留白处新增了视觉指示标记, 可辅助定位当前笔记中横幅图片的定义位置
      • 由于 Obsidian 编辑器的惰性渲染特性, 建议在文件顶部附近指定横幅图片的来源
    • 新增设置模态背景暗化强度的选项
    • 新增了ASCII艺术缩放比例选项
      • 建议编写一段CSS代码以全面自定义你的ASCII艺术作品
    • 增加了备用链接样式 (受威廉斯塔德风格启发)
    • 当未找到文件链接时, 为 Bases 添加错误处理
    • 为替换活动行号为自定义字符串设置增加了字母间距选项
    • 添加样式设置以切换TUI标注的图标是否默认不显示 (默认为 false)
    • 添加字体大小标注, 继承 UI 字体大小, 如较小、较小、中等或较大。
    • 添加了不透明度标注, 您可以一次设置标注标题、内容或两者的不透明度。
    • 添加了视图标题选项, 以指定Angled和Powerlevel10k布局中的视图标题填充
    • 以 0.05 的增量添加不透明度标注
    • 核心 Slides 插件现在在演示视图中有幻灯片编号 (最多99张幻灯片)
    • ASCII 艺术现在以核心幻灯片插件暂停演示模式出现 (按 b 触发)
    • 添加了风格化的固定图标
    • 添加了样式设置插件启动检查
    • 添加了切换显示属性标题
    • 删除了左对齐状态栏设置, 因为它太硬编码了

    变化

    • 更新了反向链接和传出链接视图的样式
    • 稍微调整了 Alternate File Tree 创建和删除对话框标题
    • 改进了 Mermaid 图的现有样式
    • 改进的侧边栏背景代码
    • 改进的 ASCII 艺术代码, 空状态操作现在默认禁用。
    • 弃用网络演示, 因为它不能很好地翻译主题的美学

    修复

    • 修复了动画, 它们现在也适用于您的笔记和侧边栏内容
    • 修复了水平规则自定义字符串在读取模式下也能工作的问题
    • 修复了 TUI 标注外观。请注意, 取消折叠大代码片段 (如标注中的 Mermaid图) 会导致渲染延迟。这适用于所有标注, 即使是默认标注。
    • 修复了在未启用选项卡标题栏的情况下显示操作系统和屏幕尺寸的问题
    • 拆分选项卡并将相关样式设置调试到自己的部分
    • 修复了扩展调色板, 以正确支持基础颜色变量的所有组合
    • 使 Bases 文本对齐一致
    • 核心幻灯片插件现在更流畅, 与主题更一致
    • 修复了默认 ASCII 艺术对齐, 可在样式设置中配置
    • 修复了突出显示中的行内数字
    • 修复了默认 TUI Callout 图标不继承图标颜色的问题
    • 修复了 Reading 模式下默认没有元数据

    1.1.x 版本

    1.1.3 标题修补

    • 修补程序损坏的标题指示器设置
    • 修补程序标题指示器与折叠图标重叠

    1.1.2 垂直修补

    • 修复了垂直布局不适用于特定 UI 区域的问题
    • 向 Bases 添加了更多有用的错误消息
    • 例如，当 Bases 查询结果为空 (找不到符合指定条件的文件), 或者找不到某个笔记的特定文件属性时
    • 为核心 Web Viewer 插件添加样式
    • 现在, 您可以为 vert-orange tui border-purple, plain-yellow 等 Callout 指定颜色。每个对应一个特定的标注样式。
    • 与 Callout 样式一样，元数据提供程序是互斥的。
    • 是写入 Callout 元数据还是作为自定义 Callout, 由您选择
    • 修复了 Callout 中的第一个 iframe 填充顶部
    • 侧边栏背景选项的性能略有提升

    1.1.1 垂直样式

    • 添加了 Callout 元数据选项, 用于在标注中设置 Lucide 图标。使用 Lucide 图标设置时, 整个 Callout 元数据或自定义标注部分必须与 Lucide 图标值匹配, 否则图标将无法呈现。
      • 例如，>[!lucide-git-fork|title-orange] 而不是 >[!note|lucide-git-fork title-orange] 或 >[!title-orange|lucide-git-fork]
      • （您不能指定多个自定义标注类型，但可以使用标注元数据进行修改）
    • 将 “垂直 LTR” Callout 元数据重命名为 “vert-ltr”
    • 添加了可选择的默认 Callout 样式。选项包括 TUI, 简化和垂直。这些还具有 Callout 元数据和自定义标注等效项。
      • 自定义标注和 Callout 元数据选项是互斥的，但可以覆盖所选的默认 Callout 样式
    • 更新社区部分外观和社区导航项目
    • 添加了可切换的垂直导航样式

    1.1.0 标注更新

    • 更改（略微降低）主题中的默认线高度
      • 您可以根据自己的喜好进行配置
    • 添加了更多详图索引样式设置选项
    • 现在, 您可以将任何有效的 Lucide 图标用作自定义标注。它将默认图标替换为您输入的清晰图标。
      • 这没有等效的标注元数据
    • 现在，您可以将所有标注元数据指定为自定义标注。例如，>[!|empty] 和 >[!empty] 都会呈现相同的样式。
    • 更新 ASCII 图标集
    • 您可以对选项卡进行编号并删除其标题（默认情况下在RTZ模式下启用）。您也可以单独切换它。
    • 添加了实验性 TUI 标注, 并提供了一个选项, 使其适用于所有标注
    • 添加了可切换的活动文件指示器
    • 样式化搜索插件
    • pl10k 布局的更多视图标题样式

    1.0.x 版本

    1.0.2 固定突出显示

    • 文本突出显示现已修复
      • 感谢 GitHub 上的 weiss-d 和 OMG Discord 上的 arthur
    • 添加了更多 Bases 插件样式选项

    1.0.1 排版更新

    • 添加了实验 Bases 插件样式设置
    • 修改了一些样式设置, 使用变量选择以获得更好的性能。保留传统选项以实现向后兼容性
    • 在应用程序启动时添加了一个实验性的加载指示器
    • 现在, 您可以配置整个主题中文本的正常文本颜色, 行高, 字母和单词间距。

    1.0.0 更新

    • 添加了 “样式设置” 选项, 用于更改状态栏对齐方式, 隐藏直到悬停动画适应当前选定的对齐方式。
      • 未选择对齐方式时, 默认为右对齐。
    • 将 “悬停前隐藏” 替换为 “悬停前保持” 选项。命名为 “隐藏直到触发”
    • 删除了水平规则自定义字符串左百分比, 因为我从 Willemstad 主题作者那里学会了如何将 <div> 居中
    • 添加了当前操作系统和屏幕尺寸的可切换显示。
      • 屏幕尺寸归功于这位开发人员和这位开发人员还有这位开发人员 。
    • 视图标题项的默认对齐方式更好。
    • 将视图标题面包屑最大宽度移动到新的视图标题部分
      • 添加了可切换选项, 以显示操作系统详细信息和屏幕尺寸, 并可配置填充
    • 将大多数与图标相关的样式设置移动到自己的部分
    • 添加了可切换的ASCII图标集
      • 笑脸图标现在默认设置为关闭
    • 添加了 WIP Iconic 插件支持, 您可以切换插件是否覆盖笑脸和 ASCII 图标
    • 修复了动画, 动画现在也显示在堆叠的选项卡中。如果它们显示在 “样式设置” 容器中, 则可以切换。
    • 将模式移动到自己的部分，由于 Obsidian Members’ Group Discord 上的 @orionp，添加了 Flex Max 模式
      • Flex Max 模式默认启用, 适用于那些不知道从哪里开始定制主题的人 (合理的默认设置)
      • 默认情况下, 一些默认启用的样式设置将 Flex Max 处理
      • 将默认情况下启用的长列表设置移动到文档中
    • 修复了 tategaki 和垂直 LTR 标注元数据中的断字间距。
    • 为编辑器顶部边距添加了样式设置选项
    • 在 Live Preview 中为代码块文件扩展名添加了自定义选项
    • 改进中文翻译和文档组织
      • 其他语言将不受支持, 因为我不了解它们, 过度依赖机器进行翻译不太好
