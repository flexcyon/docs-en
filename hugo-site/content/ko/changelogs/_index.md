#### 참고

변경 로그는 최신 버전부터 가장 오래된 버전 순으로 표시됩니다.
변경 로그를 연대순으로 보고 싶다면 사이드바나 햄버거 메뉴를 사용해 관련 하위 페이지를 확인할 수 있습니다.

---

## 버전 1.4.x
### 1.4.0 색상 재검토
#### 기능
- Bases Style Settings는 더 이상 실험적이라고 표시되지 않습니다.
- 이제 ASCII 아트와 인용문에 대해 대상 글꼴 패밀리 그룹(텍스트, 인터페이스 또는 모노스페이스)을 지정할 수 있습니다.
	- ASCII 아트가 잘 보이려면 Source Code Pro와 같은 모노스페이스 글꼴을 사용하는 것을 권장합니다.
- 이제 강조 표시 텍스트 색상과 불투명도를 구성할 수 있습니다.
- "Do not show status bar in new tab" 옵션을 추가했습니다.
	- 또는 새 탭에서 상태바의 불투명도를 설정할 수도 있습니다. 기본값은 0.55입니다.
- "Bases Card Label Opacity" 설정을 추가했으며, 기본값은 0.85입니다.
- Typewriter Mode(활성 줄 외에는 모두 어둡게)를 추가했으며, 불투명도는 기본값 0.55로 구성 가능합니다. `cssclass`를 통해 설정할 수도 있습니다.
- 이제 앱에서 지정되지 않은 경우 기본 커서 유형을 설정할 수 있습니다.
- 색상 지정된 제목이 활성화된 경우 각 제목에 사용되는 색상을 선택할 수 있습니다.
- Bases 세로 및 가로 여백 설정을 추가했으며, 기본값은 둘 다 16px입니다.
- Bases 임베드 여백을 추가했으며, 기본값은 4px입니다.
- 표 제목 및 셀 정렬 관련 설정을 추가했으며, 기존 설정을 표 제목과 셀로 분리했습니다.
- 표 셀 글꼴 크기 설정을 추가했으며, "Table Header size"를 "Table Header font size"로 이름을 변경해 더욱 구체적으로 표현했습니다. 둘 다 이제 UI 글꼴 크기 옵션 중에서 선택할 수 있습니다.
- "Do not display inline title" 설정을 추가했으며, 이에 해당하는 cssclass는 `no-inline-title`입니다.
- "Do not display view header" `cssclass`를 추가했습니다.
- 모바일 섹션과 "Revert 1.11 Navigation UI" 설정을 추가했습니다. 이 설정은 볼트 선택기와 내비게이션을 1.11 이전 스타일로 되돌립니다.

#### 변경 사항
- Style Settings 시간 단위가 s에서 ms로 변경되었습니다.
	- 영향을 받는 설정에는 대체 파일 트리, 애니메이션, 팝업 및 플래시카드 콜아웃, 마우스 호버까지 상태 숨김, 커서 지속시간 등이 포함됩니다.
- 테마는 이제 일부 파생 색상에 대해 `oklch` 색공간을 사용합니다.
- 입력 관련 설정을 '편집기 > 입력' 아래에 있는 별도 섹션으로 이동했습니다.
- 예제 트리 ASCII를 Figlet 플레이스홀더로 변경했습니다. 저작권 및 라이선스가 명확하지 않기 때문입니다.
- ASCII 아트 텍스트 정렬을 텍스트 입력 대신 옵션 드롭다운으로 변경했습니다.
- 커뮤니티 아이템 배경색을 수정했습니다.
- 색상 지정된 제목은 더 이상 PDF 내보내기 출력에 나타나지 않습니다. 인쇄(PDF 내보내기) 스타일링은 Fancy-a-Story 테마 개발자 Kapirklaa 님 덕분입니다.
- pl10k의 장치 작동 상세 정보를 수정하고, 좌측 패딩 설정을 추가했습니다. 기존 우측 패딩 설정의 기본값이 변경되었습니다.

#### 해결된 문제
- 툴팁 및 인터랙티브 액센트를 수정했습니다.
- pl10k 스타일링을 수정했습니다.
- 대체 링크 스타일링을 수정했으며, 이제 기존 링크 색상 변수를 상속하고 `oklch`를 사용합니다.
- 테마의 애니메이션 성능을 개선했습니다.
- 필요한 곳에 오버스크롤 동작을 정의했습니다.
- 코드베이스의 일부를 정리하고 수정했습니다.
- pl10k 모드 View Header의 테두리 반경을 수정했습니다.
- 리본이 표시될 때 세로 내비게이션 패딩을 수정했습니다.
- 배너 스니펫 라이브 미리보기 핫픽스

## 버전 1.3.x
### 1.3.0 애니메이션 업데이트
#### 기능
- 이제 UI의 노트와 모달과 같은 다른 부분 간에 서로 다른 애니메이션 세트를 사용할 수 있습니다.
- Flex Max 모드에서 이전의 왼쪽에서 오른쪽으로 슬라이드하는 방식 대신 기본값으로 설정된 페이드 인 애니메이션이 추가되었습니다.
- 애니메이션의 초기 불투명도와 확대 애니메이션의 초기 스케일링을 위한 Style Settings 옵션 추가
- 테마에서 배너 스니펫 지원 관련 수정사항 추가
	- 현재 노트에서 배너 이미지가 정의된 위치를 찾기 쉽게 하기 위해 라인 거터에 시각적 인디케이터가 추가되었습니다.
	- Obsidian 에디터의 지연 렌더링 특성상, 파일 상단 부근에 배너 이미지 소스를 명시하는 것이 권장됩니다.
- 모달 배경 어두워짐 강도 설정 옵션 추가
- ASCII 아트 스케일링 팩터 옵션 추가
	- 자신의 ASCII 아트를 광범위하게 맞춤화하려면 CSS 스니펫을 작성하는 것이 권장됩니다.
- 대체 링크 스타일링 추가 (Willemstad에서 영감을 받음)
- 파일에 대한 링크가 없을 때 Bases의 오류 처리 추가
- 활성 라인 번호를 사용자 정의 문자열로 교체하는 경우 자간 옵션 추가
- TUI 콜아웃의 아이콘이 기본적으로 표시되지 않도록 전환 설정 추가 (기본값은 false)
- 글꼴 크기 콜아웃 추가, UI 글꼴 크기 단계(smaller, small, medium, large)를 상속합니다.
- 불투명도 콜아웃 추가, 콜아웃 제목, 내용 또는 둘 다의 불투명도를 한 번에 설정할 수 있습니다.
- Angled 및 Powerlevel10k 레이아웃에서 View Header 제목 패딩을 지정하는 옵션 추가
- 0.05 단위로 불투명도 콜아웃 추가
- 코어 슬라이드 플러그인에 프레젠테이션 보기에서 슬라이드 번호 표시 기능 추가 (최대 99 개 슬라이드)
- ASCII 아트가 코어 슬라이드 플러그인의 일시정지 프레젠테이션 모드에서도 표시되도록 변경 (b 키 누르면 트리거됨)
- 스타일리시한 고정 아이콘 추가
- Style Settings 플러그인 체크 추가
- 속성 제목 표시 토글 추가

#### 변경 사항
- 백링크 및 아웃바운드 링크 보기의 스타일 업데이트
- 대체 파일 트리 생성 및 삭제 대화상자의 제목을 약간 조정
- 기존 머메이드 다이어그램 스타일 개선
- 사이드바 배경 코드 개선
- ASCII 아트 코드 개선, 빈 상태 액션은 이제 기본적으로 비활성화됩니다.
- 웹 데모는 테마의 미학을 잘 표현하지 못하므로 폐기되었습니다.

#### 수정 사항
- 애니메이션 수정, 이제 노트와 사이드바 콘텐츠에도 적용됩니다.
- 가로줄 사용자 정의 문자열이 읽기 모드에서도 작동하도록 수정
- TUI 콜아웃 모양 수정. 콜아웃 내에서 머메이드 다이어그램과 같은 큰 코드 스니펫을 펼치지 않으면 렌더링 지연이 발생할 수 있습니다. 이는 기본 콜아웃까지 모두 해당됩니다.
- 디스플레이 운영체제 및 화면 크기를 탭 Title Bar가 활성화되지 않은 상태에서도 작동하도록 수정
	- 탭 분할 및 디버깅 관련 Style Settings을 별도 섹션으로 이동
- 확장 색상 팔레트가 기본 색상 변수의 모든 조합을 제대로 지원하도록 수정
- Bases 텍스트 정렬을 일관되게 수정
- 코어 슬라이드 플러그인의 테마와의 호환성이 더욱 원활하고 일관되게 개선되었습니다.
- 기본 ASCII 아트 정렬을 더 합리적으로 수정, Style Settings에서 구성 가능
- 하이라이트 내 인라인 수학 수정
- 기본 TUI 콜아웃 아이콘이 아이콘 색상을 상속하지 않는 문제 수정
- 읽기 모드에서 기본 메타데이터 없음 문제 수정
- 좌측 정렬 상태바 설정이 너무 하드코딩되어 있어 제거

### 1.1.1 세로 방향 스타일링
- 콜아웃에 lucide 아이콘을 설정하는 콜아웃 메타데이터 옵션 추가. lucide 아이콘 설정을 사용할 때는 전체 콜아웃 메타데이터 또는 사용자 정의 콜아웃 섹션 모두 lucide 아이콘 값과 일치해야 하며, 그렇지 않으면 아이콘이 렌더링되지 않습니다.
	- 예: `>[!lucide-git-fork|title-orange]` 대신 `>[!note|lucide-git-fork title-orange]` 또는 `>[!title-orange|lucide-git-fork]` (여러 사용자 정의 콜아웃 유형을 동시에 지정할 수 없으며, 콜아웃 메타데이터로 수정 가능)
	- `vertical-ltr` 콜아웃 메타데이터를 `vert-ltr`로 이름 변경
- 선택 가능한 기본 콜아웃 스타일 추가. 옵션으로는 TUI, Plain 및 Vertical이 있습니다. 이들 역시 콜아웃 메타데이터 및 사용자 정의 콜아웃과 동등한 기능을 갖습니다.
	- 사용자 정의 콜아웃과 콜아웃 메타데이터 옵션은 상호 배타적이지만, 선택한 기본 콜아웃 스타일을 재정의할 수 있습니다.
- 커뮤니티 스토어 외관 및 커뮤니티 아이템 효과 업데이트
- 토글 가능한 세로 내비게이션 스타일 추가

- 이제 테마 전반에서 일반 텍스트 색상, 줄 간격, 글자 및 단어 간격을 구성할 수 있습니다.

### 1.0.0 업데이트

- 상태바 정렬 변경을 위한 Style Settings 옵션 추가, "hide until hover" 애니메이션이 현재 선택된 정렬에 맞춰 조정됩니다.
  - 정렬이 선택되지 않은 경우 기본값은 오른쪽 정렬입니다.
  - "hide until hover"를 "hold until hover" 옵션으로 대체했으며, 이름은 "hide until trigger"로 정했습니다.
- 가로줄 맞춤 문자열 왼쪽 비율을 제거했습니다. Willemstad 테마 작성자로부터 `<div>`를 가운데 정렬하는 방법을 배웠기 때문입니다 :D
- 현재 운영 체제와 화면 크기를 토글 가능한 형태로 표시하도록 추가했습니다.
  - 화면 크기 구현은 [이 개발자](https://dev.to/janeori/css-type-casting-to-numeric-tanatan2-scalars-582j), [이 개발자](https://css-tip.com/screen-dimension), [이 개발자](https://dev.to/leapcell/get-screen-size-in-pure-css-3kna) 분들 덕분입니다.
- View Header 항목의 기본 정렬을 개선했습니다.
  - View Header의 브레드크럼 최대 너비를 새로운 View Header 섹션으로 이동했습니다.
  - 운영 체제 세부 정보와 화면 크기를 토글 가능한 형태로 표시하고, 패딩도 구성 가능하게 추가했습니다.
- 대부분의 아이콘 관련 Style Settings를 별도 섹션으로 이동했습니다.
  - 토글 가능한 ASCII 아이콘 세트 추가
  - 스마일리 아이콘은 이제 기본적으로 비활성화됩니다.
  - WIP 아이코닉 플러그인 지원 추가, 플러그인이 스마일리 및 ASCII 아이콘을 덮어쓰는지 여부를 토글할 수 있습니다.
- 애니메이션을 수정했으며, 이제 스택된 탭에서도 애니메이션이 표시됩니다. Style Settings 컨테이너에서도 애니메이션이 표시되는지 토글할 수 있습니다.
- 모드를 별도 섹션으로 이동했으며, Obsidian 멤버 그룹 디스코드의 `@orionp` 덕분에 Flex Max 모드 추가
  - Flex Max 모드는 기본적으로 활성화되며, 사용자가 커스텀 설정을 어디서부터 시작해야 할지 모르는 분들을 위한 것입니다(편향된 기본값).
  - 기본적으로 활성화되었던 일부 Style Settings는 기본적으로 비활성화되고 Flex Max에서 처리됩니다.
  - 기본적으로 활성화되는 설정들의 긴 목록을 문서로 이동했습니다.
- `tategaki` 및 `vertical-ltr` 콜아웃 메타데이터의 잘못된 자간을 수정했습니다.
- 에디터 상단 여백을 위한 Style Settings 옵션 추가
- Live Preview에서 코드 블록 파일 확장자를 위한 맞춤 설정 옵션 추가
- 중국어 번역과 문서 구성을 개선했습니다.
  - 다른 언어는 지원하지 않습니다. 제가 모르는 언어이고, 번역을 너무 기계에 의존하는 것은 좋지 않기 때문입니다.
